Gene expression data for WormBase ParaSite

Introduction

About us

We’re an EBI - Sanger collaboration project, and we run a resource with genomes of parasitic worms.

We rely on the Ensembl web code and Ensembl Genomes pipelines, together with customizations that make sense for our data and users.

Value of gene expression data to our users

  • Confirming gene models

  • Hints at gene function

  • Putative drug targets need to be expressed in the relevant life stage

The need for custom solution instead of Expression Atlas widget

Data mostly deposited to ENA, which Expression Atlas doesn’t curate.

Sequencing parasites is difficult and there isn’t much funding for it, so we need to be more inclusive in curation.

The data will be worse, so we need to be more conservative in analysis.

RNASeq-er as source of data

Also by EBI Functional Genomics Team.

Analyses all RNASeq runs in ENA - alignment and quantification

Our solution

Alignment tracks

Aggregated into a UCSC-style track hub. We also offer JBrowse as an alternative genome browser, and they mostly went there.

Details

Data we offer

  • read counts and transcripts per million reads (TPMs) for each gene in each ENA run
  • TPMs in a condition – median value for all runs representing the same condition
  • differential expression analysis between selected pairs of conditions

Metadata

There are 584 studies available from RNASeq-er for our worms. I selected and curated metadata for 125/279 studies apart from C. elegans, and 29 out of 305 for C. elegans.

Semi-automatic based on automatic criteria, with extensive checks. Stored in files, managed with git.

Software

All written by me and Tuan in very nice Perl.

Simple techniques:

  • metadata is curated as files, and stored with source code
  • the studies page is a single HTML document per species
  • results are in TSV files, easy to download from the studies page
  • data for the gene page is constructed by running grep on TSV files

We didn’t need:

  • databases
  • Hive: there’s one entry point to the whole program
  • JavaScript

Future directions

We will keep the dataset updated as more data comes out - estimating 10 studies per release, so it’s not much work to keep it up to date.

Incremental improvements: more informative studies page, differential expression data in a better format, etc.
